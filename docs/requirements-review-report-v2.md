## 🟡 Should Fix

### S-R1. FR-03 与 NFR-10 对公开读取接口的豁免范围表述不一致

**问题描述**：
- FR-03 写的是"所有业务 API（**除登录外**）必须验证 JWT token 的有效性"
- NFR-10 写的是"所有业务 API（**除公开读取接口外**）必须携带有效 JWT token"
- 实际设计中，车型数据（FR-14 P0 "无需鉴权"）和色卡/材质数据（Section 7.1 标注"公开读取"）不需要鉴权
- FR-03 的"除登录外"只豁免了登录接口，未豁免公开读取接口，与 FR-14 和实际 API 设计矛盾

**为什么需要修复**：开发人员若只看 FR-03（不交叉对照 NFR-10），可能错误地对车型和色卡查询接口加上 JWT 鉴权，导致客户端不登录就无法加载车型列表——这是业务不可用的。

**修改建议**：将 FR-03 修改为"所有业务 API（除登录和公开读取接口外）必须验证 JWT token 的有效性（签名、过期时间）"，与 NFR-10 对齐。
